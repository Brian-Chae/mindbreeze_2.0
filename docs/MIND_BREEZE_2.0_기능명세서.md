## 1. 시스템 개요

MIND BREEZE 2.0은 상담사·명상 지도사용 B2B SaaS 웹 서비스로, 다음 4개 도메인을 통합한다.

1. **신뢰성 확보** — 상담센터·상담사 자격 자동 검증
2. **상담 운영** — 내담자 관리·세션 예약·채팅
3. **AI 기록·리포트** — 음성→STT→요약→리포트
4. **뇌파 측정 (선택)** — LINK BAND EEG 실시간 모니터링·분석

# F1. 계정·인증·온보딩

## F1.1 회원 가입 (역할 선택)

- **액터**: Guest
- **설명**: 가입 시 `상담사`/`내담자` 중 역할 선택. 역할에 따라 후속 온보딩이 분기.
- **메인 플로우**:
  1. 이메일·비밀번호 입력 (또는 소셜 로그인)
  2. 이메일 인증 (6자리 OTP, 10분 유효)
  3. 역할 선택 카드 (상담사 / 내담자)
  4. 약관·개인정보·민감정보 처리 **분리 동의**
  5. 역할별 온보딩 진입
- **예외**: 이메일 중복 / OTP 만료 → 재발송 가능 (60초 쿨다운)
- **입력**: email, password(8자↑, 영문+숫자+특수), role
- **출력**: userId, accessToken, refreshToken
- **수락 기준 (QA)**:
  - [ ] 비밀번호 Argon2id 해시 저장 확인
  - [ ] OTP 10분 후 만료, 재발송 60초 쿨다운
  - [ ] 민감정보 동의 미체크 시 가입 진행 불가
  - [ ] 동일 이메일 중복 가입 차단
- **우선순위**: P0 / **개발 단계**: MVP1

## F1.2 로그인 / 로그아웃 / 토큰 갱신

- **메인 플로우**: 이메일·비밀번호 → JWT(15분) + Refresh(14일) 발급
- **수락 기준**:
  - [ ] 5회 연속 실패 시 15분 잠금
  - [ ] Refresh Token 회전(Rotation) 적용, 탈취 시 invalidate
  - [ ] 로그아웃 시 서버측 Refresh 폐기
- **P0 / MVP1**

## F1.3 상담사 온보딩 4단계

- **Step 1**: 기본 정보(이름·이메일·연락처)
- **Step 2**: 상세 정보(성별·생년월일·경력연수·전문 분야)
- **Step 3**: **소속 형태 + 자격 증빙** (→ F2 / F3 연결)
- **Step 4**: 프로필(사진·한줄 소개) + AI 검증 진행 안내
- **완료 화면**: 상담사 코드 6자리 발급 + 인증 등급 뱃지
- **수락 기준**:
  - [ ] Step 3 미완료 시 가입 보류(`Unverified`)
  - [ ] 검증 진행 중에도 본인 대시보드는 접근 가능, 내담자 매칭은 불가
- **P0 / MVP1**

## F1.4 내담자 온보딩 4단계

- **Step 1~3**: 기본·상세·프로필
- **Step 4**: **상담사 지정** — 상담사 코드 6자리 입력(OTP 스타일)
- **수락 기준**:
  - [ ] 잘못된 코드 입력 시 명확한 오류 + 재입력
  - [ ] 매칭된 상담사가 `Verified` 미만이면 매칭 거부
- **P0 / MVP1**

## F1.5 비밀번호 재설정

- 이메일 링크(30분 유효) → 새 비밀번호 입력
- 직전 3개 비밀번호 재사용 차단
- **P1 / MVP1**

# F2. 상담센터 등록·인증

## F2.1 기존 센터 검색 후 소속 가입

- **액터**: Counselor (가입 중)
- **플로우**:
  1. 센터명·지역으로 검색 (자동완성)
  2. 센터 선택 → 가입 신청 발송
  3. 해당 센터 `OrgAdmin`에게 알림 (이메일)
  4. `OrgAdmin` 승인 → 상담사의 `orgId` 확정
- **수락 기준**:
  - [ ] 중복 신청 차단
  - [ ] 거절 시 사유 표시, 다른 센터로 변경 가능
- **P0 / MVP1**

## F2.2 신규 센터 등록 (대표자)

- **플로우**:
  1. 센터명·대표자명·사업자등록번호·주소·연락처 입력
  2. **사업자등록증 업로드** (PDF/JPG/PNG, ≤10MB)
  3. **AI 자동 검증** (→ F3.5 파이프라인)
     - Gemini OCR → 필드 추출
     - 국세청 사업자 상태 조회 API 대조
     - 위변조 탐지
  4. 판정: `auto_approve` / `needs_review` / `reject`
  5. 승인 시 신청자가 `OrgAdmin` 역할 자동 부여
- **수락 기준**:
  - [ ] 사업자등록번호 형식·체크섬 검증
  - [ ] 국세청 응답 "휴업/폐업" 시 자동 반려
  - [ ] OCR 결과와 사용자 입력 불일치 시 `needs_review`
  - [ ] 증빙 파일 S3 SSE-KMS 저장, 단기 프리사인드 URL
- **P0 / MVP1**

## F2.3 1인 / 프리랜서 운영

- 별도 센터 없이 진행. `orgId = null` 허용.
- 상담사 개인 자격 증빙(F3)만으로 인증.
- **P0 / MVP1**

## F2.4 센터 정보 수정 / 상담사 관리

- **액터**: OrgAdmin
- 소속 상담사 목록·승인·해제, 센터 프로필 편집
- **P1 / MVP1**

# F3. 상담사 자격 증명

## F3.1 자격 증빙 업로드

- **업로드 종류**:
  - `license` (국가/민간 자격증)
  - `diploma` (학위·졸업증명)
  - `career` (경력증명)
  - `id_card` (신분증)
- **포맷**: PDF/JPG/PNG, ≤10MB/건, 최대 10건
- **수락 기준**:
  - [ ] 신분증은 필수, 자격증·학위·경력은 최소 1개 이상 필수
  - [ ] 업로드 시 S3 SSE-KMS, 메타만 PG `Credential`
- **P0 / MVP1**

## F3.2 인증 등급 산정

- 등급: `Verified+` / `Verified` / `Self-Declared` / `Unverified`
- 산정 규칙은 종합 기획 **§3A.5** 따름 (`Verified+` / `Verified` / `Self-Declared` / `Unverified`)
- **수락 기준**:
  - [ ] `Verified` 미만은 내담자 검색·매칭 결과에서 제외
  - [ ] 등급 변동 시 사용자에게 이메일 통지
- **P0 / MVP1**

## F3.3 인증 뱃지 표시

- 프로필 카드·검색 결과·예약 화면에 등급 뱃지 노출
- **P0 / MVP1**

## F3.4 자격 만료 관리

- `expiresAt` 30일 전 자동 리마인드, 만료 시 해당 자격 비활성
- 갱신 자료 재업로드 → F3.5 재검증
- **P2 / MVP3**

## F3.5 AI 자동 검증 파이프라인 (핵심)

- **호출 시점**: 증빙 업로드 직후 비동기 Celery 태스크
- **단계**:
  1. **OCR/필드 추출** — Gemini Vision (멀티모달)
  2. **문서 유형 분류** — Gemini 프롬프트 분류
  3. **공공 API 대조** — 사업자 상태, 자격 발급기관 DB(가능 범위)
  4. **위변조 탐지** — 픽셀 일관성·폰트·인장 휴리스틱 + LLM 검사 → `risk_score`
  5. **교차 일관성** — 신분증 이름 ↔ 자격증 ↔ 학위 ↔ 사용자 입력
  6. **판정** — 규칙 엔진:
     - `auto_approve`: 공공 API 일치 + risk<0.15 + 모든 일관성 통과
     - `needs_review`: 임계값 중간 또는 공공 API 미존재
     - `reject`: 명백한 위변조·불일치
  7. **사용자 안내** — 반려 사유 LLM 초안 + 재제출 가이드
- **출력**: `Credential.aiVerdict` JSONB + `VerificationAudit` 로그
- **수락 기준**:
  - [ ] 각 단계 결과는 모두 감사 로그에 저장
  - [ ] 외부 API 호출 시 식별자 최소 전송(주민번호 뒷자리 마스킹 등)
  - [ ] 모델 응답 5초 SLA 초과 시 큐에 적재, UI는 "검토 중"
  - [ ] `reject` 시 동일 파일 재업로드 차단(해시 기반)
- **P0 / MVP1**

## F3.6 본인 자격 대시보드 (`My Credentials`)

- **P0 / MVP1**

  - [ ] 네트워크 끊김 시 로컬 버퍼링 후 재전송
  - [ ] 최대 녹음 길이 3시간
- **P0 / MVP1**

## F7.2 STT + 화자 분리 (Gemini)

- 세션 종료 → Celery 태스크 → **Gemini API (Audio)**
- 출력: `[{speaker, text, start, end}, ...]`
- **수락 기준**:
  - [ ] 30분 음성 기준 5분 내 STT 완료(목표, NFR-03)
  - [ ] 화자 라벨 정확도 ≥80% (한국어 1:1 대화 기준)
  - [ ] Gemini 실패 시 Whisper+pyannote 폴백 (`docs/AI_STACK_DECISION.md`)
- **P0 / MVP1**

## F7.3 AI 기록지 (요약·기록 초안)

  - 최면: 단계별 변화·유도 기법
  - 명상: 흐름·이완 추이
- **수락 기준**:
  - [ ] 초안 생성 후 상담사가 자유 편집 가능
  - [ ] "편집됨" 표시 + 변경 이력

# F9. LINK BAND EEG 연동

## F9.1 상담사 PC Web Bluetooth 연결

- **사전 조건**: Chrome/Edge, HTTPS, 사용자 제스처
- 디바이스 스캔 → GATT Service UUID 필터 → 연결
- **수락 기준**:
  - [ ] Safari/Firefox에서 진입 시 안내 메시지
  - [ ] 연결 실패 사유별 메시지(권한·범위·디바이스)
  - [ ] 재연결 자동 시도(최대 3회)
- **P0 / MVP2**

## F9.2 EEG 스트림 수신·저장

- BLE Characteristic 구독 → WebSocket → 서버
- 서버: 버퍼링 → 세션 종료 시 **S3 Parquet** 업로드
- 메타(`durationSec`, `sampleRate`)는 PG `EEGRecord`
- **수락 기준**:
  - [ ] 250Hz 기준 30분 데이터 무손실 저장
  - [ ] 연결 끊김 구간은 빈 값 + 플래그
- **P0 / MVP2**

## F9.3 실시간 대시보드 (1:1)

- 시계열 차트(원파형) + 지표 게이지(집중도/이완도/스트레스)
- **P0 / MVP2**

## F9.4 EEG 분석 (오프라인)
