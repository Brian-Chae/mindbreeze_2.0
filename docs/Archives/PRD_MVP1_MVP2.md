# MIND BREEZE — MVP1·MVP2 통합 PRD

**제품 요구사항 문서 (Product Requirements Document)**  
버전 1.0 | 2026.03 | 원본: `MIND_BREEZE_Service_Plan.md`  
범위: **MVP1(핵심 기반) + MVP2(AI 기록·리포트)** 단일 릴리스 구현

---

## 문서 목적

본 문서는 MIND BREEZE 확장 플랫폼의 **첫 번째 통합 릴리스**에 포함될 기능·비기능 요구사항을 정의한다. 기획서의 MVP1·MVP2 범위를 **한 번에 구현**할 때 개발·QA·기획이 동일한 기준으로 정렬할 수 있도록 한다.

---

## 1. 제품 개요

### 1.1 비전 (요약)

심리상담사·명상 지도사의 업무 효율을 **AI 기반 자동 기록·요약**으로 높이고, 선택적으로 **LINK BAND 2.0** 뇌파 측정을 결합해 내담자 상태를 과학적으로 파악할 수 있는 **웹 기반 통합 멘탈 케어 플랫폼**. LINK BAND 없이도 예약·기록·AI 요약·리포트 등 핵심 가치를 제공한다.

### 1.2 본 릴리스(MVP1+MVP2)의 목표

| 목표 | 설명 |
|------|------|
| **G1** | 상담사/지도사가 세션을 생성·관리하고 내담자를 초대·관리할 수 있다. |
| **G2** | 선택적으로 LINK BAND를 연결해 1:1 세션에서 실시간 뇌파를 모니터링할 수 있다. |
| **G3** | 세션 음성을 녹음·STT·화자 분리 후 LLM으로 요약·기록지를 자동 생성한다. |
| **G4** | 상담사용·내담자용 리포트를 생성·조회·발송할 수 있다. |
| **G5** | (선택) 명상 수업 시 다수 참여자를 그리드로 모니터링할 수 있다. |

### 1.3 서비스 범위 (기획서 정합)

- **세션 유형:** 임상 심리상담(1:1) / 최면 심리상담(1:1) / 명상 수업(다수)
- **플랫폼:** 웹 (Web Bluetooth API)
- **하드웨어:** LINK BAND 2.0 — **선택(opt-in)**

---

## 2. 범위 정의

### 2.1 포함 범위 (In Scope) — MVP1 + MVP2

#### MVP1 — 핵심 기반

1. 사용자 인증(회원가입/로그인), 역할 구분(최소: 전문가 vs 내담자/참여자)
2. 세션 CRUD, 세션 유형 3종, 상태(예정/진행중/완료/취소)
3. 내담자/참여자 프로필·초대·예약 확인 플로우
4. Web Bluetooth 기반 LINK BAND 연결(선택), EEG 스트림 수신
5. 실시간 뇌파 모니터링 대시보드 — **1:1 상담용 단일 뷰**(선택)
6. 기본 EEG 분석 지표 시각화(선택) — 집중도/이완도/스트레스

#### MVP2 — AI 기록 및 리포트

1. 음성 녹음 및 STT 파이프라인
2. 화자 분리(Speaker Diarization)
3. AI 세션 요약 및 기록지 자동 생성(LLM)
4. 상담사용 세션 리포트 생성·조회
5. 내담자용 개인화 리포트 생성·발송
6. **(선택)** 명상 수업 다자 모니터링 그리드 뷰

### 2.2 제외 범위 (Out of Scope) — MVP3 이후

다음은 **본 PRD 릴리스에 포함하지 않는다.** (기획서 MVP3)

- 셀프 트레이닝 프로그램 처방·콘텐츠(가이드 명상/호흡 등) 전체 제품화
- 일일 트레이닝 누적 분석 및 상담 세션과의 심층 연계 뷰
- 예약/알림 시스템의 고도화(Email/Web Push를 **필수**로 하는 전체 알림 인프라)
- 대시보드 통계·추이 분석·내담자 타임라인 **고도화**(MVP1~2 수준의 기본 화면만 해당 시 포함)

> **알림:** MVP1 기획에 “예약 확정 시 알림”이 있으나 MVP3에 상세 예약/알림이 별도 항목으로 있다. 본 통합 릴리스에서는 **최소 알림(예: 인앱·이메일 중 팀이 선택한 하나)** 또는 **수동 공유 링크**로 충족 가능 여부를 [7. 가정 및 오픈 이슈](#7-가정-및-오픈-이슈)에서 확정한다.

## 6. 기술 제약·아키텍처 정합

기획서 [6. 기술 아키텍처]를 기본 준수한다.

| 계층 | 기술 (기획서 기준) |
|------|---------------------|
| Frontend | React + TypeScript, TanStack Query, Zustand, Recharts/D3 |
| BLE | Web Bluetooth API |
| Realtime | WebSocket (Socket.IO) |
| Backend | FastAPI, Celery + Redis, SQLAlchemy + Alembic |
| DB | PostgreSQL + TimescaleDB(시계열 EEG) |
| Storage | S3 호환(음성·PDF) |
| AI | STT(Whisper/Clova 등), Diarization(pyannote 등), LLM(Claude/GPT), EEG 분석(룩시드랩스 SDK) |

**핵심 엔티티:** User, Session, EEGRecord(선택), SessionRecord, Report, TrainingProgram/TrainingLog는 **MVP3**까지 확장 예정이나, 본 릴리스에서 **SessionRecord·Report** 스키마는 반드시 설계한다.

## 7. 가정 및 오픈 이슈

제품·일정 확정을 위해 아래를 스프린트 0에서 결정한다.

1. **알림 채널:** Email만 / Push만 / 인앱만 / 조합 — MVP3 전체 알림과의 경계
2. **반복 일정:** MVP1~2에서 주간/월간 반복을 구현할지, 수동 복제로 대체할지
3. **명상 수업 그리드(FR-EEG-07):** P2로 두었으나 동시 접속·라이선스·성능 예산 확정
4. **EEG 분석 지표:** 룩시드랩스 SDK 출력 필드와 UI 라벨 매핑 표
5. **STT/LLM 벤더:** 비용·한국어 품질·온프레미스 여부

## 8. 성공 지표 (제안)

| 지표 | 정의 | 목표 방향 |
|------|------|-----------|
| 세션 완료율 | 생성된 세션 중 완료 비율 | 증가 |
| 녹음·요약 성공률 | STT+요약 작업 성공 비율 | 임계값 이상 |
| LINK BAND 사용률 | opt-in 세션 비율(선택 지표) | 서비스 정책에 따름 |
| 리포트 발송률 | 완료 세션 대비 리포트 발송 비율 | 증가 |
| 전문가 NPS/만족도 | 사내/파일럿 설문 | 기준선 대비 개선 |

## 10. 사용자 플로우 요약 (수용 기준 참고)

- **전문가:** 로그인 → 세션 생성·초대 → (선택) LINK BAND 안내 → 녹음 동의 → 진행 중 모니터링/마커 → 종료 → AI 요약·기록지 편집 → 리포트 승인·발송
