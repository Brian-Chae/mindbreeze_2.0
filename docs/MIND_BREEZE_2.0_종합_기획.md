## 1. 서비스 개요

**MIND BREEZE 2.0**은 룩시드랩스의 LINK BAND 뇌파 측정 기술과 AI 분석 엔진을 결합한 **뇌파 기반 심리상담·명상 통합 서비스 플랫폼**이다.

- **비전:** 심리상담사와 명상 지도사의 업무 효율을 AI 기반 자동 기록·요약으로 혁신하고, 선택적 LINK BAND 연동으로 내담자의 상태를 과학적으로 파악할 수 있는 통합 멘탈 케어 플랫폼.
- **서비스 모델:** B2B (상담사/명상 지도사가 고객, 내담자는 최종 사용자)
- **플랫폼:** 웹 (관리자 + 내담자) + 모바일 웹앱 (내담자)
- **하드웨어:** LINK BAND 2.0 — **선택적 사용 (opt-in)**
- **서비스 플로우:** 진단(설문) → 케어(과제 수행) → 리포트(상담 결과) — AI가 전 과정 지원

### 1.1 핵심 차별점

| 구분 | 기존 명상 앱 | MIND BREEZE 2.0 |
|------|------------|-----------------|
| 측정 | 자가 보고 | LINK BAND 실시간 뇌파 측정 |
| 기록 | 수기/메모 | AI 자동 녹음·STT·요약 |
| 피드백 | 일반적 가이드 | 개인화 AI 리포트 + 뇌파 분석 |
| 운영 | 수동 관리 | 통합 예약·내담자·리포트 대시보드 |
| 하드웨어 | 불필요 | LINK BAND 선택적 사용 (미착용 시에도 핵심 기능 제공) |

## 2. 설계 철학 (5대 원칙)

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **LINK BAND는 선택이다** | 모든 핵심 기능(예약·기록·AI 요약·리포트)은 LINK BAND 없이도 완전히 동작한다. 뇌파 측정은 부가 가치. |
| 2 | **AI가 기록한다, 사람이 결정한다** | STT·요약·리포트 초안은 AI가 생성하되, 최종 승인·편집은 상담사의 권한. |
| 3 | **데이터 프라이버시는 설계 기반** | 뇌파·상담 데이터는 기본적으로 암호화. 동의 기반 수집. 접근 로그 필수. |
| 4 | **웹 우선, 모바일 확장** | Web Bluetooth API로 브라우저에서 직접 LINK BAND 연동. 네이티브 앱 없이 시작. |
| 5 | **B2B SaaS, B2C 경험** | 상담사/기관이 비용을 지불하는 B2B 모델. 내담자는 무료로 초대받아 프리미엄 경험. |

## 3. 사용자 유형 및 페르소나

### 3.1 전문가 (상담사 / 명상 지도사)

**역할:** 세션 생성·운영·기록 관리·리포트 발행의 주체

| 유형 | 세션 형태 | LINK BAND 권장도 | 핵심 니즈 |
|------|----------|:---:|------|
| **임상 심리상담사** | 1:1 상담 | 중간 | 내담자 심리 상태 추적, 상담 기록 자동화 |
| **최면 심리상담사** | 1:1 최면 | **높음** | 의식/무의식 전환 구간 모니터링 |
| **명상 지도사** | 그룹 수업 | **높음** | 다수 참여자 동시 상태 파악, 수업 효과 측정 |

### 3.2 클라이언트 (내담자 / 참여자)

**역할:** 세션 참여, 동의 시 LINK BAND 착용, 리포트 확인, 셀프 트레이닝 수행

| 유형 | 참여 형태 | LINK BAND | 핵심 니즈 |
|------|----------|:---:|------|
| **상담 내담자** | 1:1 예약 | 선택 | 개인 리포트, 상담사와의 소통, 셀프 트레이닝 |
| **명상 참여자** | 그룹 수업 | 선택 | 수업 일정 확인, 그룹 리포트, 셀프 명상 |

## 5. 핵심 서비스 플로우

### 5.1 세션 라이프사이클

```
[예정] → [진행중] → [완료] → [리포트 발행]
   ↓         ↓         ↓
[취소]   [일시정지]  [재개]
```

### 5.2 상담사 여정 (Phase 1-3)

**Phase 1 — 세션 준비**
1. 로그인 → 대시보드 (오늘 예약·알림·통계)
2. 새 세션 생성: 유형 선택(임상/최면/명상) → 일정 → 내담자 초대
3. 이전 세션 기록 확인: 기록지·셀프 트레이닝 데이터

**Phase 2 — 세션 진행**
4. 세션 시작: LINK BAND 연결 안내(선택) → 음성 녹음 동의
5. 실시간 모니터링: LINK BAND 연결 시 뇌파 대시보드 / 미연결 시 타이머만
6. 주요 시점 마커: 특이사항 타임스탬프 태그 + 메모

**Phase 3 — 세션 종료 및 후속**
7. 세션 종료: 녹음 종료 → AI 요약 자동 생성 (수 분 내)
8. 기록지 확인·편집 → 저장
9. 리포트 발행: 내담자용 리포트 확인 → 발송 승인
10. 셀프 트레이닝 처방 (MVP3)

### 5.3 내담자 여정

1. 초대 알림 수신 → 웹 접속 → 회원가입/로그인
2. 예약 확인 → 세션 참여 → (선택) LINK BAND 연결
3. 세션 후 리포트 수신·확인
4. 셀프 트레이닝 수행 (MVP3)

## 6. LINK BAND 연동 아키텍처

### 6.1 연결 플로우

```
① 사용자 제스처 → BLE 스캔 시작
       ↓  (navigator.bluetooth.requestDevice)
② LINK BAND GATT Service UUID 필터링
       ↓
③ EEG Characteristic 구독 (startNotifications)
       ↓
④ Raw EEG 데이터 WebSocket → 백엔드 전송
       ↓
⑤ 백엔드 AI 분석 → 결과 WebSocket → 프론트엔드 반환
```

### 6.2 기술 제약

| 항목 | 내용 |
|------|------|
| 지원 브라우저 | Chrome, Edge 등 Chromium 기반 |
| 미지원 브라우저 | Safari, Firefox — 안내 UX 표시 |
| 모바일 | Android Chrome 지원 / iOS Safari 미지원 → Capacitor 셸 대응 (MVP2) |
| BLE 버전 | Web Bluetooth API (BLE 5.0) |

### 6.3 LINK BAND 착용 여부에 따른 기능 차이

| 기능 | 미착용 (기본) | 착용 (Opt-in) |
|------|:---:|:---:|
| 세션 예약·관리 | ✅ | ✅ |
| 음성 녹음·STT | ✅ | ✅ |
| AI 요약·기록지 | ✅ (텍스트 기반) | ✅ (뇌파 연동) |
| 실시간 뇌파 대시보드 | ❌ | ✅ |
| 뇌파 분석 리포트 | ❌ | ✅ |
| 셀프 트레이닝 | ✅ (가이드만) | ✅ (뇌파 피드백) |

## 7. AI 파이프라인

```
음성 녹음 → STT 변환 → 화자 분리 → LLM 요약 → 리포트 생성
                     (Whisper/Clova)  (pyannote)   (Claude/GPT)

                        ↓ (LINK BAND 착용 시)

                Raw EEG → EEG 분석(SDK) → 뇌파 지표 → 리포트 통합
```

### 7.1 AI 구성 요소

| 단계 | 기술 | 역할 |
|------|------|------|
| STT | Whisper API / Clova Speech | 한국어 음성 → 텍스트 변환 |
| 화자 분리 | pyannote.audio | 상담사/내담자 발화 구분 |
| 상담 요약 | Claude/GPT API | 세션 요약·기록지·다음 제안 |
| EEG 분석 | 룩시드랩스 AI SDK | 집중도/이완도/스트레스 지수 |
| 리포트 생성 | LLM + 템플릿 엔진 | 개인화 PDF 리포트 |

### 7.2 세션 유형별 AI 요약 항목

| 세션 유형 | 요약 항목 | 기록지 포함 |
|----------|----------|-----------|
| **임상 심리상담** | 주요 호소 사항·감정 변화·상담사 개입 포인트 | 세션 요약·뇌파 분석(선택)·다음 제안·메모 |
| **최면 심리상담** | 최면 단계별 뇌파 변화·의식/무의식 전환·유도 기법 | 최면 과정·뇌파 그래프(선택)·다음 권고 |
| **명상 수업** | 수업 흐름·참여자 평균 이완도·구간별 효과 | 수업 콘텐츠·참여자 데이터(선택)·효과 통계 |

## 8. 기술 아키텍처

### 8.1 시스템 구성도

| 계층 | 기술 | 역할 |
|------|------|------|
| Frontend | React 18 + TypeScript, Vite | SPA 구조, 대시보드 UI, 실시간 차트 |
| BLE 연동 | Web Bluetooth API | LINK BAND BLE GATT 연결·EEG 스트림 |
| Realtime | Socket.IO | 실시간 뇌파·채팅·세션 상태 |
| Backend | FastAPI (Python) | REST API, 인증, 세션·내담자 관리 |
| Async Queue | Celery + Redis | STT 변환·AI 요약·리포트 생성 |
| Database | PostgreSQL + TimescaleDB | 사용자/세션(OLTP) + 시계열 EEG |
| Storage | S3 호환 | 음성 녹음·리포트 PDF·미디어 |
| AI/ML | Looxid SDK + LLM API | EEG 분석 + 상담 요약 |

### 8.2 기술 스택 상세

**Frontend**
- React 18+ (TypeScript) — SPA
- TanStack Query — 서버 상태·캐싱
- Zustand — 클라이언트 상태 (BLE 연결·실시간 데이터)
- Recharts / D3.js — 뇌파 차트·리포트 시각화
- Web Bluetooth API — LINK BAND BLE
- Web Audio API — 브라우저 음성 녹음

**Backend**
- FastAPI (Python) — REST API + WebSocket
- Celery + Redis — 비동기 작업 큐
- Socket.IO — 실시간 EEG·채팅
- SQLAlchemy + Alembic — ORM·마이그레이션

### 11.1 전체 타임라인

| 단계 | 핵심 목표 | 기간 | 주요 산출물 |

| **MVP 2** | AI 기록·리포트 | 6주 | 녹음·STT·화자분리·LLM 기록지·리포트 |

| **총계** | **전체 플랫폼 완성** | **20주 (5개월)** | — |

### 11.2 MVP 1 — 핵심 기반 (8주)

1. 사용자 인증 (회원가입·로그인·역할 구분)
2. 세션 CRUD (3종 유형·상태 관리)
3. 내담자/참여자 관리 (프로필·초대·예약)
4. Web Bluetooth LINK BAND 연결(선택) — 페어링·EEG 스트림
5. 실시간 뇌파 모니터링 대시보드(선택) — 1:1 상담 단일 뷰
6. 기본 EEG 분석 지표 시각화(선택) — 집중도/이완도/스트레스

### 11.3 MVP 2 — AI 기록·리포트 (6주)

1. 음성 녹음 + STT 변환 파이프라인
2. 화자 분리 (Speaker Diarization)
3. AI 세션 요약·기록지 자동 생성 (LLM)
4. 상담사용 세션 리포트 생성·조회
5. 내담자용 개인화 리포트 생성·발송
6. 명상 수업 다자 모니터링 그리드 뷰(선택)

### 11.4 MVP 3 — 셀프 트레이닝·고도화 (6주)

1. 셀프 트레이닝 프로그램 관리 (상담사 처방)
2. 가이드 명상·호흡 훈련 콘텐츠
3. 일일 트레이닝 기록·누적 추이 분석
4. 상담 세션 ↔ 셀프 트레이닝 데이터 연계
5. 예약·알림 시스템 (Email/Web Push)
6. 대시보드 고도화 (통계·추이·내담자 타임라인)

## 15. 오픈 이슈

1. **알림 채널:** Email vs Push vs 인앱 — MVP3 전체 알림과의 경계 설정 필요
2. **반복 일정:** MVP1에서 주간/월간 반복 구현 여부
3. **STT/LLM 벤더:** 비용·한국어 품질·온프레미스 여부 결정
4. **채팅 방식:** 자체 구축 vs 외부 채널(카카오 등) 연동
5. **iOS BLE:** 웹 한정 vs Capacitor 네이티브 셸 시점 결정
6. **LINK Cloud SDK 연동:** 분석 API 스펙·라이선스 협의
7. **멀티테넌트:** MVP1에서 단일 기관 vs 즉시 멀티테넌트 설계
8. **의료기기 인증:** LINK BAND 규제 범위 확인 (진단/치료 목적 아님)
